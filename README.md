## 🚀 Plan de Trabajo Detallado por Semana

### Semana 1: El Corazón de los Datos (SQL & Data Modeling)

El puesto exige "Optimización avanzada". No basta con saber `SELECT`.

- **Lunes-Miércoles:** Práctica `WINDOW FUNCTIONS`, `CTEs` y optimización de índices.
- **Jueves-Viernes:** Crea un modelo de datos estrella (Star Schema) para un caso de transacciones bancarias.
- **Fin de Semana:** Proyecto 1: Diseña una base de datos desde cero que detecte duplicados en registros de clientes (CDD).

### Semana 2: La Nube y Escalabilidad (Snowflake & AWS)

Debes entender cómo se mueven los datos a gran escala.

- **Lunes-Miércoles:** Snowflake. Aprende sobre "Time Travel", "Cloning" y carga de datos desde archivos planos.
- **Jueves-Viernes:** Ecosistema AWS. Enfócate en **S3** (donde vive el Data Lake) y **Athena** para consultar esos datos.
- **Fin de Semana:** Proyecto 2: Simula la carga de un CSV de transacciones a una tabla de Snowflake usando comandos `COPY INTO`.

### Semana 3: El Motor de la Eficiencia (ETL & Python)

Aquí es donde te diferencias automatizando lo que otros hacen a mano.

- **Lunes-Miércoles:** Usa Python (Pandas) para limpiar datos "sucios" de servicios financieros (fechas inconsistentes, nombres mal escritos).
- **Jueves-Viernes:** Crea un Pipeline básico: Extraer (CSV) -> Transformar (Python) -> Cargar (SQL).
- **Fin de Semana:** Proyecto 3: Automatiza un reporte que identifique transacciones sospechosas por encima de $10,000 automáticamente.

### Semana 4: Visualización e Inteligencia (BI & AI)

El toque final para presentar resultados a la dirección.

- **Lunes-Martes:** Power BI avanzado. Crea medidas en DAX para calcular el riesgo promedio de un cliente.
- **Miércoles-Jueves:** Introducción a IA. Entiende cómo los modelos de Machine Learning pueden predecir el fraude (detección de anomalías).
- **Viernes:** Repaso de "Soft Skills": Liderazgo técnico y resolución de problemas financieros.
- **Fin de Semana:** Simulación de entrevista técnica y pulido de portafolio.

## 💡 Tips de tu Coach para el Éxito

1. **Enfoque en Finanzas:** No estudies datos genéricos. Siempre usa ejemplos de **bancos, cuentas y transacciones**.
2. **Documenta tu Proceso:** En la entrevista, no solo digas que sabes SQL; explica cómo optimizaste una consulta que tardaba minutos a segundos.
3. **Calidad sobre Cantidad:** El rol exige "atención al detalle" y "control de calidad". Asegúrate de que tus ejercicios siempre incluyan pasos de validación de datos.

Este plan es intensivo, pero cubrirá los pilares de **Data Analysis, Data Science y AI** aplicados al mundo real de los delitos financieros.
